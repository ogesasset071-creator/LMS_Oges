LMS OGES APPLICATION FLOWS AND FEATURE INVENTORY
Ground truth based on current frontend pages, backend routes, and database schema in this codebase.

Purpose
- This document lists the implemented flows and features for Learner and Admin/Educator users.
- It also includes items that exist in UI or code but are not fully wired, so nothing is overstated when sharing with a client.
- This file can be converted into PDF later.

==================================================
1. ROLE STRUCTURE IN THE APPLICATION
==================================================

The application currently has 4 role surfaces:
- Learner
- Educator
- Admin
- Super Admin

Important role behavior:
- Learners are routed to the normal learner home flow.
- Super Admin has a separate frontend route at `/super-admin`.

3. LEARNER FLOW AND FEATURES
==================================================

3.1 Learner home page flow
- Home page hero for internal training portal
- Recommended courses section based on learner category
- Category roadmap section tied to learner specialization
- Roadmap has level-style progression blocks
- PP-based lock logic on some recommended/roadmap content
- Featured category section for SaaS and Oil and Gas
- Continue Training section for learner’s in-progress courses
- Explore All Training Tracks section
- Category tabs
- Level tabs
- Course cards with title, thumbnail, category, level, tutor/instructor name
- Direct navigation from cards into player page

3.2 Course discovery flow
- Courses page with:
- keyword search
- category filter
- pagination / load more
- course card listing
- chapter count display
- derived difficulty tag
- start learning button

- Categories page exists in routing and is intended as a category discovery page.
- Tutor/Admin listing page exists for discovering experts/leads.

3.3 Admin / lead discovery from learner side
- Learner can browse admins/leads page
- Search leads by name, role, or bio
- Filter by broad category labels
- View lead cards with avatar, name, role, and bio
- Open tutor/lead profile page from list

3.4 Learner course player flow
- Open course via `/player/:courseId`
- Fetch course details from backend
- Fetch learner’s saved notes for that course
- Auto-select first chapter or first chapter in first unit
- Embedded video playback via YouTube iframe or direct URL
- Sidebar tabs:
- Content
- Notes
- Resources

Content tab features
- Units and chapters listing
- Chapter switching
- Active chapter highlighting
- Chapter title display
- Chapter description display
- Tutor name display
- Category display

Progress and engagement behavior inside player
- Auto heartbeat progress update while watching
- First watch/start record after delay
- Backend can mark lesson progress records
- XP increases on progress events
- PP increases on progress events
- Total minutes increase on progress events
- “Add to My Training” button adds course to learner dashboard by creating progress on current chapter

Certificate / completion flow from player
- PP badge shows learner PP against course required PP
- Certificate action is locked until required PP threshold is met
- When learner confirms certificate action:
- backend marks all chapters completed for that course
- learner receives additional XP
- completion emails can be sent
- learner is redirected to dashboard

Notes flow
- Learner can write private notes per course
- Notes are loaded from backend
- Notes are saved back to backend

Resources flow
- Learner can view chapter-level resources
- Learner can view course-level resources
- Learner can open/download materials from resource links

Personal resource flow
- Learner can upload private study materials for a course
- Learner provides custom file title
- Learner chooses file and uploads it
- Learner can later view uploaded private materials list for that course

5. ADMIN VS SUPER ADMIN DIFFERENCE
==================================================

Admin
- Uses the live `/admin` dashboard
- Handles training operations
- Can create and manage courses/trainings
- Can create assignments and quizzes
- Can post course announcements
- Can review submissions and completions

Super Admin
- Uses separate route `/super-admin`
- Has a different UI from the normal admin/educator dashboard
- Is currently a separate frontend panel
- Current implementation is mock/demo style and not connected to live backend data

Key distinction
- `Admin` and `Super Admin` are not the same role.
- `Admin` belongs to the operational training-management dashboard.
- `Super Admin` belongs to a separate high-level panel.

8. ITEMS PRESENT IN CODE BUT NOT FULLY WIRED OR NOT RELIABLE
==================================================

These points are important if this document is being sent to a client as an exact feature statement.

Leaderboard
- Backend leaderboard API exists.
- Leaderboard page component exists.
- Footer links to `/leaderboard`.
- But `App.jsx` does not currently register a `/leaderboard` route.
- Result: leaderboard is coded, but not fully reachable from routing as-is.

Tutor profile published-course list
- Tutor profile page calls `/tutors/:id/courses`.
- No matching backend route was found.
- Result: tutor profile course listing is not fully wired.

Educator stats in profile
- Profile page calls `/educator/stats`.
- Backend route found is `/api/user/admin/stats`, not `/educator/stats`.
- Result: educator insights on profile are likely not loading correctly.

My Assignments for admin
- Educator dashboard fetches `/user/educator/assignments`.
- No matching backend GET route was found.
- There is a duplicated backend decorator issue around `/admin/assignments/{id}`.
- Result: admin “My Assignments” list is not reliable in current code.

Assignment update from admin
- Educator dashboard calls PUT `/admin/assignments/:id` for editing.
- No backend PUT route for assignment update was found.
- Result: edit assignment flow is not fully implemented server-side.

Categories page
- Route exists and page exists, but it was not necessary for core learner/admin flow validation here.
- It should be treated as a discovery/support page, not the primary source of training data.

Assignments page create modal
- Frontend includes a generic create-assignment modal in learner-facing assignments page.
- It posts to `/assignments`, but no matching backend POST route was found for that path.
- Actual admin assignment creation is handled in admin dashboard via `/admin/assignments`.

Some backend admin checks
- A few backend functions use `current_user.role` instead of `current_user.Lms_role`.
- That can break some educator/admin endpoints depending on execution path.
